BANCO DE DADOS AULA 03

INTRODUÇÃO -> Aborda o uso de subqueries e funções de decisão para manipular
e formatar dados 

->Subqueries são comandos SELECT aninhados para obter resultados combinados

->Funções de decisões são usadas para formatar os resultados ou para lidar 
com possiveies falhas

SUBQUERIES ->Uma subquery é um comando SELECT dentro de outro SELECT , usada 
para satisfazer uma cluasula WHERE

->A subquery (inner query) é geralmente executada antes da consulta principal
com uma clausula WHERE que contém outra consulta SELECT

EXEMPLO DE SUBQUERIES

->Exemplos práticos mostram como usar subqueries para encontrar funcionarios 
com salários maiores que o de "abel"

->E para identificar funcionarios com o mesmo cargo de outro funcionário 
especifico 

->Subqueries podem ser usadas para comparar cargos e para selecionar funcionarios
do mesmo departamento

->Subqueries podem ser usadas para selecionar empregados que ganham acima 
da média salarial

TIPOS DE SUBQUERIES

.SINGLE ROW SUBQUERY ->Retorna um único valor, como no exemplo de encontrar 
funcionários nos departamentos de marketing e administração

.SUBPESQUISAS ANINHADAS->Encontrar empregados cujo salário é maior que o maior
salário no departamento 'sales'

OPERADORES DE COMPARAÇÃO PARA MULTIPLAS LINHAS

->A unidade aborda operadores como IN, ANY, ALL e EXISTS para comparar valores
com resultados de subqueries

->O operador ANY compara um valor com cada valor retornado pela subquery

->O operador ALL compara um valor com todos os valores retornados pela subquery

FUNÇÕES DE DECISÃO: DECODE

->A função DECODE facilita pesquisas condicionais ,funcionando como uma ferramenta
"se-então-se não"

->A sintaxe é DECODE (col/expressão, procurado1, resultado1,...,padrão),onde a
coluna/expressão é comparada com valores procurados

->Se nenhum valor procurado for encontrado , a função DECODE retorna o valor
padrão ou um valor nulo

FUNÇÕES DE DECISÃO: CASE

->A função condicional CASE permite processamento condicional com multiplas
hipóteses

->A estrutura básica é CASE WHEN condição1 THEN resultado1 WHEN condição2 THEN 
resultado2 ELSE resultado_padrão END
