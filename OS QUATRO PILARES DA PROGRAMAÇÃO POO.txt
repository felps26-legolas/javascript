OS QUATRO PILARES DA PROGRAMAÇÃO ORIENTADA A OBJETOS 

OS QUATRO PILARES ->Abstração , encapsulamento , herança e polimorfismo

ABSTRAÇÃO -> Na POO damos o nome de abstração para o processo de aproximar o mundo
real do mundo da programação, sendo seu objetivo:

->Simplificar um problema dificil, para isso a abstração leva em conta os aspectos
importantes de um determinado ponto de vista e desconsidera os aspectos restantes

->A abstração é a criação de uma classe abstrata , que é uma classe incompleta,
como se fosse um quebra cabeça

->Assim , esta classe não permite a criação de instancias e obriga a implementação
de todos os métodos da classe assinados como abstract

->Resumindo a abstração se concentra apenas nas informações que são importantes
para o seu propósito 

->Concentrando-se apenas no que é importante para uma determinada finalidade

->Abstração nada mais é que você abstrair coisas , quando você tem algo muito
grande, mas não há necessidade de cadastrar todas as informações

ENCAPSULAMENTO -> Ao encapsular algo , você está colocando um objeto dentro de um recipiente 
igual a um remédio de capsula

->É uma das principais técnicas da programação orientada a objetos

->Quando você encapsula um objeto , você está criando uma proteção e um padrão

->O propósito é de proteger o desenvolvedor do código e o código do desenvolvedor

->Assim quando você encapsula algo na POO, você está criando moldes padrão que fazem
com que o conteudo do objeto não importe

->Você está determinando que o resultado será sempre o mesmo

->É a açã de ocultar partes independentes da implementação, permitindo construir
partes invisiveis ao mundo exterior

CONCEITO DE INTERFACE -> É uma lista de serviços fornecidos por um componente

->É o contato com o mundo exterior, que define o que pode ser feito com um objeto
de determinada classe

->Ou seja a interface é uma classe composta apenas por métodos (não tem atributos)

ATENÇÃO -> Encapsular não é obrigatório na POO, mas é uma ótima prática para 
produzir classes mais eficientes

HERANÇA -> A herança na POO também diz respeito à ação de herdar

->Nada mais é do que um obejto pode ser criado em uma outra classe 

->levando consigo todos os atributos já existentes em sua classe de origem

->É uma maneira de reutilizar o código já existente em uma nova classe, o código é
aprimorado com novas capacidades

POLIMORFISMO -> É um pilar da POO que é utilizado para que duas classes façam o uso
do mesmo método, implementando-o de formas diferentes

->Permite que o programador desenvolva o código de forma ampla ao invés de perder
muito tempo no desenvolvimento de códigos especificos
