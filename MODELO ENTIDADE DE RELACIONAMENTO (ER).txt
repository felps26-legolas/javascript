INTRODUÇÃO AO MODELO - ER

-> Modelo ER (entidade de relacionamento) é uma tecnica utilizada para modelagem de dados
(heuser, 2010)

->Esse modelo foi proposto por peter pin-shan chen, na década de 1970, descreve o modelo 
conceitual de um banco de dados

-> definimos entidade como "conjunto de objetos da realidade modelada sob os quais se deseja
manter informações no banco de dados"

->no modelo ER, a estrutura de um banco de dados é retratada como um diagrama, chamado de 
diagrama de entidade-relacionamento(DER)

OS PRINCIPAIS ELEMENTOS DE UM DER SÃO:

.ENTIDADE -> Uma classe de objetos do mundo real com caracteristica e propriedades comuns sobre
as quais desejamos registrar informaçoes 

.RELACIONAMENTO -> Uma associação entre duas ou mais entidades

.ATRIBUTOS -> Uma caracteristica de uma entidade ou relacionamento 

.No DER, as entidades são representadas por retangulos

.Relacionamentos são representados por losangos 

.Atributos por circulos

.O DER auxilia os analistas no que diz respeito à concepção, otimização e desenvolvimento
de um projeto de banco de dados

AS VANTAGENS DO MODELO-ER:

->Excepcional simplicidade conceitual
->Representação visual
->ferramenta de comunicação eficaz
->integrado com modelo de banco de dados relacional

AS DESVANTAGENS

->Representação limitada de restrições 
->representação limitada de relacionamento

->Diagramas de ER constituem um quadro muito útil para a criação e manipulação de bases de
dados

->Esses diagramas são de fácil compreenção e não requer treinamento para trabalhar com ele 
eficientemente

->significa que os projetistas podem usar diagramas de ER para se comunicar com desenvolvedores

->os diagramas de ER são prontamente traduziveis em tabelas relacionais que podem ser usadas
para construir bases de dados mais rápido

->os diagramas de ER podem ser aplicados em outros contextos como descrever as diferentes
relações e operações dentro de uma organização

->Uma entidade é um objeto interessantepara o usuario final e refere-se realmente ao conjunto
de entidades e não a uma única entidade

->A entida no MER corresponde a uma tabela

->Uma entidade pode ser uma pessoa, lugar , evento ,onjeto ou conceito q é relevante para um
determinado sistema para o qual estamos modelando o projeto de banco de dados 

EXEMPLO

PESSOA -> Estudante , Empregado , Cliente

LUGAR -> Cidade, Sala , Armazem

EVENTO -> Festa, Casamento, Exposição, Feira

OBJETO -> Carro, Navio, Máquina , Produto

CONCEITO -> Projeto ,Conta , Curso

Essas entidades são representadas no DER por um retangulo e nomeadas, usando substantivos
singulares , ex, em um sistema escolar podemos ter as seguintes entidades 
[aluno]   [professor] [curso]
[turma]   [histórico]

DISTINÇÃO ENTRE ENTIDADE , INSTÂNCIA DE ENTIDADE E CONJUNTO DE ENTIDADES

ENTIDADE -> Define uma coleção de entidades que possuem os mesmos atributos

INSTÂNCIA DE ENTIDADE -> É um único item nesta coleção

CONJUNTO DE ENTIDADES -> É umconjunto de instancias de entidade

EXEMPLO -> ALUNO é um tipo de entidade, um aluno com o número de RGM "5384236" é uma instancia
de entidade , Uma coleção de todos os alunos é um conjunto de entidades

Existem três tipos de entidades , fortes , fracas e associativas

ENTIDADES FORTES -> Se uma entidade pode existir separadamente de todas as suas entidades 
relacionadas, essa é classificada como entidade forte

->além disso ainda possuem independencia de identificação dada por um ou mais atributos 
próprios

ENTIDADE FRACA -> É uma entidade que depende da existencia de outra entidade

->pode ser definida como uma entidades que não pode ser identificada por seus próprios 
atributos 

-> Entidade fraca é aquela que atende a duas condições 

1 -> É dependente existencia de uma instancia da entidade com a qual tem um relacionamento

2 -> Tem uma chave primária que é parcial ou totalmente derivada da entidade-mãe no relacionamento

ENTIDADE ASSOCIATIVA

-> Havendo informações sobre o relacionamento, essas informações serão mantidas em uma 
associativa

EXEMPLO -> O número de horas em que o empregado trabalhou em um projeto especifico é um
atributo da relação entre empregado e projeto , não de empregado ou de projeto

-> ima entidade associativa é identificada unicamente por concatenação das chaves primárias
das duas entidades que ele conecta
